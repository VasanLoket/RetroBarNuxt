# Retrowave Bar — Веб-приложение на Nuxt 3

Современный веб-сайт для бара с ретро-атмосферой.  
Реализована полная клиент-серверная архитектура: регистрация/вход, меню с категориями, корзина заказов, бронирование столиков, личный кабинет и система лояльности.

---

## Стек технологий

| Компонент          | Технология                                      |
|--------------------|-------------------------------------------------|
| **Фронтенд**       | Nuxt 3 (Vue 3, Composition API, SSR/SSG)        |
| **Стили**          | CSS (глобальный + scoped)                       |
| **База данных**    | SQLite (лёгкая, файловая)                       |
| **Авторизация**    | bcrypt (хеширование паролей), JWT (не используется, но данные в localStorage) |
| **Сервер**         | Nuxt Nitro (встроенный сервер)                  |
| **Деплой**         | Vercel / Netlify (для статики) или любой Node.js хостинг |

## Установка и настройка

### 1. Клонирование репозитория
```bash
git clone https://github.com/VasanLoket/RetroBarNuxt.git
cd RetroBarNuxt
```

Если вы получили архив, распакуйте его и перейдите в папку проекта.

### 2. Установка зависимостей
```bash
npm install
```
Это установит все необходимые пакеты: Nuxt, sqlite3, bcrypt и другие.

### 3. Создание базы данных и заполнение тестовыми данными
Проект использует SQLite, который создаёт файл `server/db/menu.db`. Выполните:
```bash
node server/db/seed.js
```
Вы увидите сообщение об успешной инициализации.  
В базе создадутся таблицы `menu` (блюда) и `users` (пользователи), а также добавится тестовое меню.

### 4. Запуск сервера разработки
```bash
npm run dev
```
После запуска откройте в браузере `http://localhost:3000`.

## Основные функции приложения

### Авторизация
- Регистрация нового пользователя (логин и пароль).
- Вход в аккаунт.
- Пароли хранятся в БД в виде хеша (bcrypt).
- Состояние пользователя сохраняется в `localStorage`.

### Меню
- Загрузка данных из БД (SQLite) через серверный API `/api/menu`.
- Группировка блюд по категориям.
- Режим просмотра / режим заказа (счётчики количества).

### Корзина
- Добавление/удаление блюд.
- Изменение количества.
- Подсчёт итоговой суммы.
- Сохранение корзины в `localStorage`.

### Бронирование столиков
- Интерактивная схема зала.
- Клик по столику → открывается модалка с датой и временем (Для заглушки используем текущее время + 1 час + случайные минуты).

### Личный кабинет
- Отображение логина и количества баллов.
- История начислений/списаний (заглушка, можно расширить).
- Выход из аккаунта.

## Деплой

### Вариант 1: Статический сайт (для Vercel/Netlify)
```bash
npm run generate
```
Папка `.output/public` содержит все статические файлы. Загрузите их на ваш хостинг.

### Вариант 2: Серверный рендеринг (SSR) на своём сервере
```bash
npm run build
node .output/server/index.mjs
```
Для продакшена используйте PM2 для управления процессом.

## Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| `npm install` долго или падает с ошибкой | Проверьте интернет. Если ошибка с `bcrypt`, установите `windows-build-tools` (Windows) или `build-essential` (Linux). |
| Ошибка SQLITE при создании БД | Удалите `server/db/menu.db` и выполните `node server/db/seed.js` заново. |
| Страница меню пустая | Проверьте, что API `/api/menu` возвращает данные (зайдите в браузере на `http://localhost:3000/api/menu`). Если ошибка, перезапустите сервер. |
| Авторизация не работает | Убедитесь, что БД содержит таблицу `users`. Проверьте консоль браузера на ошибки. |
| Стили не применяются | Подключите глобальный CSS в `nuxt.config.ts` (уже должно быть). |
